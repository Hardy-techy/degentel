# Degentel LP 🔮

Degentel LP is an institutional-grade DeFi Infrastructure Node built on the **CROO Agent Protocol**. 

While most Web3 AI agents focus on basic token safety (e.g., honeypot checks), serious DeFi capital requires deep mathematical forensics before providing liquidity to Automated Market Makers (AMMs). Degentel LP provides deterministic, on-chain analytics and Impermanent Loss projections for Yield Farmers, DAOs, and AI Trading Bots.

## The 3 Core Services

Degentel LP exposes 3 distinct services to the CROO Network:

### 1. IL Forecaster 📉
Predicts Impermanent Loss risk before capital is deployed.
* **How it works:** Pulls 30 days of OHLCV data for a specific Liquidity Pool to calculate the historical standard deviation (volatility).
* **Deliverable:** Projects the exact dollar-value loss per $1,000 deposited, annualized volatility, and assigns a strict Risk Classification.

### 2. Deep Liquidity Audit 🔎
Evaluates the true financial health and profitability of an AMM Pool.
* **How it works:** Extracts deep on-chain metrics including Fully Diluted Valuation (FDV), 24-hour Buy/Sell ratios, unique transactors, and pool age.
* **Deliverable:** Calculates the exact Capital Efficiency Score (Volume/TVL) to estimate the true yearly APY generated by trading fees.

### 3. Yield Route Finder 🛣️
Discovers the highest-yielding deployment routes for a specific token.
* **How it works:** Scans all Decentralized Exchanges (DEXes) on a given network for a target token.
* **Deliverable:** Returns a ranked array of the top 3 Liquidity Pools by volume and estimated APY, allowing automated trading bots to instantly route capital to the most profitable smart contract.

## Technical Architecture
- **CROO Node.js SDK**: Natively integrates with the CROO Agent Protocol for trust-minimized escrow and delivery.
- **Dynamic Routing**: Uses `order.serviceId` to natively map dashboard UI orders to backend logic without requiring users to input service types.
- **GeckoTerminal Data Engine**: Pulls deterministic, real-time blockchain data across multiple networks (Ethereum, Base, Solana).

## Getting Started (Local Deployment)

1. Clone the repository and install dependencies:
```bash
npm install
```

2. Configure your `.env` file with your CROO SDK Key and Service IDs:
```env
CROO_API_URL=https://api.croo.network
CROO_WS_URL=wss://api.croo.network/ws
CROO_SDK_KEY=your_sk_key_here

# Service IDs (From CROO Dashboard)
CROO_SERVICE_IL_FORECAST=srv_123
CROO_SERVICE_AUDIT=srv_456
CROO_SERVICE_ROUTE_FINDER=srv_789
```

3. Start the Agent Provider:
```bash
node src/index.js
```
*The agent will connect to the CROO WebSocket and begin listening for incoming on-chain orders.*
